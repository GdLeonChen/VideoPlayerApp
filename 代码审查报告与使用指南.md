## 问题修复详情

### #1: 手势方向错误 🔴

**问题描述**
原代码中 `swipeUp` 被定义为"切换到上一个视频"，与用户习惯相反（下滑 = 下一个视频）。

**修复前**
```javascript
if (this.lastSwipeY - currentY > 100) {
    // 向上滑动 → 上一个视频 (错误)
    this.playPreviousVideo();
}
```

**修复后**
```javascript
if (this.lastSwipeY - currentY > 100) {
    // 向上滑动 → 下一个视频 (符合直觉)
    this.playNextVideo();
} else if (currentY - this.lastSwipeY > 100) {
    // 向下滑动 → 上一个视频
    this.playPreviousVideo();
}
```

**影响范围**: `VideoPlayer.js` - `_handleSwipe()` 方法

---

### #2: 未监听视频结束事件 🔴

**问题描述**
视频播放结束时，播放器没有自动切换到下一个视频，用户体验不连贯。

**修复前**
```javascript
this.$video.addEventListener('play', () => {
    this.$controlBar.classList.add('playing');
});
```

**修复后**
```javascript
this.$video.addEventListener('play', () => {
    this.$controlBar.classList.add('playing');
});

// 新增：监听视频结束事件
this.$video.addEventListener('ended', () => {
    this.playNextVideo();
});
```

**影响范围**: `VideoPlayer.js` - `_bindVideoEvents()` 方法

---

### #3: Android 13+ 权限缺失 🟡

**问题描述**
Android 13 (API 33) 引入新的媒体权限 `READ_MEDIA_VIDEO`，替代旧的 `READ_EXTERNAL_STORAGE`。原代码仅声明了旧权限，导致 Android 13+ 设备无法访问视频文件。

**修复前**
```xml
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
```

**修复后**
```xml
<!-- Android 13+ 新权限 -->
<uses-permission android:name="android.permission.READ_MEDIA_VIDEO" />
<!-- 兼容旧版本 -->
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"
                 android:maxSdkVersion="32" />
```

**影响范围**: `VideoPlayerApp/app/src/main/AndroidManifest.xml`

---

### #4: 滚动事件内存泄漏 🟡

**问题描述**
`wheel` 事件监听器未在播放器销毁时移除，导致多次切换视频后累积大量监听器。

**修复前**
```javascript
document.addEventListener('wheel', (e) => {
    if (e.deltaY > 0) {
        this.playNextVideo();
    } else {
        this.playPreviousVideo();
    }
});
```

**修复后**
```javascript
// 绑定时保存引用
this._wheelHandler = (e) => {
    if (e.deltaY > 0) {
        this.playNextVideo();
    } else {
        this.playPreviousVideo();
    }
};
document.addEventListener('wheel', this._wheelHandler);

// 新增销毁方法
destroy() {
    document.removeEventListener('wheel', this._wheelHandler);
    // ...其他清理逻辑
}
```

**影响范围**: `VideoPlayer.js` - 构造函数 + 新增 `destroy()` 方法

### #5: 控制栏显示逻辑混乱 🟡

**问题描述**
点击视频区域时，`_toggleControlBar()` 会切换控制栏状态，但与进度条拖动、音量调节等手势冲突。

**修复前**
```javascript
this.$video.addEventListener('click', () => {
    this._toggleControlBar();
});
```

**修复后**
```javascript
// 引入拖拽状态标记
this.isDragging = false;

// 只在非拖拽状态下切换控制栏
this.$video.addEventListener('click', (e) => {
    if (!this.isDragging) {
        this._toggleControlBar();
    }
});

// 在拖拽开始时设置标记
this.$progressWrap.addEventListener('touchstart', () => {
    this.isDragging = true;
});
```

**影响范围**: `VideoPlayer.js` - `_bindControlBarEvents()` + 进度条相关方法

---

### #6: 进度条不支持触摸拖拽 🟡

**问题描述**
原进度条仅支持点击跳转，不支持拖拽，用户体验差。

**修复前**
```javascript
this.$progressWrap.addEventListener('click', (e) => {
    const rect = this.$progressWrap.getBoundingClientRect();
    const ratio = (e.clientX - rect.left) / rect.width;
    this.$video.currentTime = ratio * this.$video.duration;
});
```

**修复后**
```javascript
_bindProgressEvents() {
    this.isDraggingProgress = false;

    // 触摸开始
    this.$progressWrap.addEventListener('touchstart', (e) => {
        e.stopPropagation();
        this.isDraggingProgress = true;
        this.$progressWrap.classList.add('dragging');
        this._seekByRatio(this._getRatioFromTouch(e.touches[0]));
    }, { passive: true });

    // 触摸移动
    this.$progressWrap.addEventListener('touchmove', (e) => {
        if (this.isDraggingProgress) {
            this._seekByRatio(this._getRatioFromTouch(e.touches[0]));
        }
    }, { passive: true });

    // 触摸结束
    this.$progressWrap.addEventListener('touchend', () => {
        this.isDraggingProgress = false;
        this.$progressWrap.classList.remove('dragging');
    });
}

_getRatioFromTouch(touch) {
    const rect = this.$progressWrap.getBoundingClientRect();
    return Math.max(0, Math.min(1, (touch.clientX - rect.left) / rect.width));
}

_seekByRatio(ratio) {
    const time = ratio * this.$video.duration;
    this.$video.currentTime = time;
    this._updateProgress();
}
```

**影响范围**: `VideoPlayer.js` - 新增三个方法

### #7: NAS 功能未实现 🟢

**问题描述**
UI 已提供 NAS 服务器配置入口，但缺少实际 HTTP 请求逻辑。

**修复前**
```javascript
async loadFromNAS(url) {
    console.log('NAS URL:', url);
    // TODO: 实现实际加载逻辑
}
```

**修复后**
```javascript
async loadFromNAS(url) {
    try {
        const response = await fetch(url);
        if (!response.ok) throw new Error(`HTTP ${response.status}`);

        const contentType = response.headers.get('Content-Type');
        if (!contentType?.startsWith('video/')) {
            throw new Error('非视频文件');
        }

        const blob = await response.blob();
        const videoUrl = URL.createObjectURL(blob);
        this.playVideo(videoUrl, 'NAS 视频');

        // 清理旧 URL
        if (this.currentVideoUrl) {
            URL.revokeObjectURL(this.currentVideoUrl);
        }
        this.currentVideoUrl = videoUrl;
    } catch (error) {
        alert(`加载失败: ${error.message}`);
        console.error('NAS 加载错误:', error);
    }
}
```

**影响范围**: `VideoPlayer.js` - `loadFromNAS()` 方法

## APK 安装指南

### 环境要求

- **Android 版本**: 5.0 (API 21) 或更高
- **存储空间**: 至少 10 MB 可用空间
- **权限**: 存储权限（首次启动时授予）

### 安装步骤

#### 方法一：直接安装（推荐）

1. 将 `视频播放器.apk` 文件复制到手机存储
2. 在手机文件管理器中找到 APK 文件，点击安装
3. 根据系统提示允许"未知来源应用"安装
4. 安装完成后，点击"打开"启动应用

#### 方法二：通过 ADB 安装（开发者）

```powershell
# 1. 连接手机到电脑，开启 USB 调试
# 2. 进入 APK 所在目录
cd "C:\Users\admin\Downloads\TMALL or JD tag"

# 3. 执行安装命令
adb install 视频播放器.apk

# 4. 如果已安装，使用覆盖安装
adb install -r 视频播放器.apk
```

### 验证安装

- 检查手机桌面是否有"视频播放器"图标
- 点击图标启动，应显示黑色播放器界面
- 首次启动时，系统会请求存储权限，点击"允许"

## 应用使用说明

### 功能模块图示

```
┌─────────────────────────────────┐
│                                 │
│      [顶部状态栏] 隐藏状态       │
│                                 │

│ [本地文件] [NAS设置]            │
└─────────────────────────────────┘
```

### 操作指南

#### 1. 播放本地视频

1. 点击底部"本地文件"按钮
2. 系统打开文件选择器，导航到视频所在文件夹
3. 选择一个或多个视频文件（支持多选）
4. 应用自动加载第一个视频并开始播放

#### 2. 切换视频

- **向上滑动屏幕**: 切换到下一个视频
- **向下滑动屏幕**: 切换到上一个视频
- **视频结束自动切换**: 当前视频播放完毕后，自动播放下一个

#### 3. 进度控制

- **点击进度条**: 跳转到指定位置
- **拖拽进度条**: 按住进度条滑动，实时预览

#### 4. 音量和亮度

- **右侧上下滑动**: 调整音量
- **左侧上下滑动**: 调整屏幕亮度

#### 5. 配置 NAS 视频

1. 点击底部"NAS 设置"按钮
2. 输入 NAS 视频的 HTTP 地址，例如：
   ```
   http://192.168.1.100:8080/videos/movie.mp4
   ```
3. 点击"加载"按钮开始播放

**注意事项**:
- 确保手机与 NAS 在同一局域网
- NAS 需要配置 HTTP 文件服务或 DLNA 媒体服务器
- 部分 NAS 需要用户认证，暂未实现（v3.0 计划支持）

#### 6. 全屏控制

- 点击视频区域任意位置：显示/隐藏控制栏
- 应用默认全屏沉浸式模式，自动隐藏状态栏

### 依赖冲突解决

**问题**: `androidx.activity:activity:1.9.0` 引入了 `kotlin-stdlib-jdk8:1.6.21`，与 `kotlin-stdlib:1.8.22` 冲突。

**解决方案**:

在 `app/build.gradle` 中添加：
```gradle
configurations.all {
    resolutionStrategy {
        force 'org.jetbrains.kotlin:kotlin-stdlib:1.8.22'
        force 'org.jetbrains.kotlin:kotlin-stdlib-common:1.8.22'
    }
    exclude group: 'org.jetbrains.kotlin', module: 'kotlin-stdlib-jdk7'
    exclude group: 'org.jetbrains.kotlin', module: 'kotlin-stdlib-jdk8'
}
```

在 `gradle.properties` 中添加：
```properties
kotlin.stdlib.default.dependency=false
```

### 3. 性能优化方向

#### 3.1 WebView 优化

**当前问题**: WebView 内存占用较高，多视频切换后可能卡顿。

**优化方案**:
```java
// 启用硬件加速
webView.setLayerType(View.LAYER_TYPE_HARDWARE, null);

// 禁用不必要的功能
webView.getSettings().setCacheMode(WebSettings.LOAD_NO_CACHE);
webView.getSettings().setRenderPriority(WebSettings.RenderPriority.HIGH);

// 释放内存
@Override
protected void onDestroy() {
    super.onDestroy();
    webView.loadUrl("about:blank");
```

### 常见问题 FAQ

**Q1: 安装后无法打开，提示"解析包错误"？**

A: 进入系统设置 → 应用管理 → 视频播放器 → 权限 → 存储 → 允许。

**Q3: NAS 视频加载失败？**  
A: 检查以下几点：
- 确保手机和 NAS 在同一局域网
- 确认 NAS 已开启 HTTP 文件服务
- 检查 URL 格式是否正确（如 `http://192.168.1.100:8080/video.mp4`）

**Q4: 播放卡顿或黑屏？**  
A: 尝试：
- 关闭后台其他应用
- 确保视频编码格式兼容（推荐 H.264）
- 检查手机存储空间是否充足
